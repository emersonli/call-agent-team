# Team Orchestrator Skill

为多 Agent 框架智能匹配专业团队的编排技能。支持应用研发、政府行政等多种团队类型，采用渐进式披露设计原则。

## ✨ 核心特性

- **多领域支持**: 应用研发、政府行政（三省六部制）、企业运营等
- **智能领域识别**: 根据任务描述自动匹配最合适的团队类型
- **渐进式披露**: 通用流程 + 领域特定配置，按需加载知识
- **精细化角色定义**: 每个角色包含能力模型、职责边界、最佳实践
- **动态规模计算**: 考虑预算、时间、质量等约束条件智能调整
- **协作模式推荐**: 敏捷冲刺、快速响应、三省六部制工作流程

## 🎯 支持的团队类型

### 1. 应用研发团队 (`dev-team`)

**适用场景**: 软件开发、网站建设、移动应用、系统集成

**典型场景配方**:
| 场景 | 工期 | 预算 | 团队配置 |
|-----|------|------|---------|
| MVP 开发 | 2-4 周 | 5-15 万 | 1 产品 +1 前端 +1 后端 +0.5 测试 |
| 完整应用 | 2-3 月 | 20-50 万 | 1 产品 +2 前端 +2 后端 +1 测试 +1 设计 |
| 大型系统 | 3-6 月 | 50-200 万 | 2 产品 +3 前端 +3 后端 +2 测试 +1 运维 +1 架构师 |

**协作模式**: 敏捷冲刺、快速响应

### 2. 政府行政团队 (`government-team`) ⭐ 新增！

**适用场景**: 政策制定、行政管理、公共事务、法规出台

**组织架构**: 三省六部制
- **中书省**: 决策起草（相当于政策研究室）
- **门下省**: 审议封驳（相当于人大法工委 + 审计署）
- **尚书省**: 最高执行（相当于国务院）
- **六部**: 吏部（人事）、户部（财政）、礼部（外交教育）、兵部（国防）、刑部（司法）、工部（建设）

**典型场景配方**:
| 场景 | 工期 | 参与部门 | 特点 |
|-----|------|---------|------|
| 政策制定 | 4-8 周 | 中书省 + 门下省 + 相关部委 | 决策、审议、执行三分离 |
| 官员任免 | 2-4 周 | 吏部 + 中书省 + 门下省 | 考察、推荐、审核、任命 |
| 大型工程 | 3-12 月 | 工部主导 + 户部 + 兵部等 | 跨部门协作，资金 + 人力保障 |
| 外交接待 | 1-2 周 | 礼部主导 + 中书省 | 礼仪、翻译、文书 |

**工作流程**: 提案 → 起草 → 审议 → 批准 → 执行 → 监督

## 💡 渐进式披露设计

本技能采用**渐进式披露**原则，避免信息过载：

### Level 1: 通用知识（SKILL.md 前部）
- 核心工作流程
- 通用决策矩阵  
- 基本调用方法
- **适合**: 日常使用和快速了解

### Level 2: 领域概览（SKILL.md 中部）
- 各领域快速参考卡片
- 典型场景和配置速查
- 领域特色说明
- **适合**: 选择合适团队类型时参考

### Level 3: 详细配置（YAML 配置文件）
- 完整的组织架构
- 详细的角色定义
- 具体的工作流程
- **适合**: 深度定制和扩展

### Level 4: 最佳实践（reference.md + examples.md）
- 深度案例分析
- 常见问题解答
- 优化建议
- **适合**: 解决复杂问题和疑难场景

**优势**:
- ✅ 非研发任务不会看到大量研发术语
- ✅ 按需加载对应领域的详细知识
- ✅ 保持 SKILL.md 简洁（<300 行）
- ✅ 易于扩展新领域而不影响现有结构

## 🤝 贡献指南

### 添加新团队类型

1. 在 `configs/teams/` 下创建新的 YAML 文件
2. 定义组织架构、场景配方、角色库
3. 在 `SKILL.md` 的快速决策矩阵中添加该类型
4. 在 `examples.md` 中添加使用示例

### 优化渐进式披露

- 保持 SKILL.md 通用部分简洁
- 领域特定内容放入独立配置文件
- 使用清晰的层级和导航
- 为非专业用户提供快速参考卡片
